# Домашнее задание к занятию «Безопасность в облачных провайдерах»
# Задание 1: Статический сайт в Yandex Cloud с шифрованием KMS и HTTPS

## Цель работы
Создать статический сайт в Yandex Cloud Object Storage, зашифровать содержимое бакета с помощью ключа KMS (AES-256) и обеспечить доступ к сайту по протоколу HTTPS.

## Выполненные шаги

### 1. Создание KMS-ключа
Был создан симметричный ключ `s3-bucket-key` с алгоритмом шифрования `AES-256`.
```
yc kms symmetric-key create --name s3-bucket-key --default-algorithm aes-256 (06.png, 08.png)
Результат:
ID ключа: abj0db3sjm76h6icdbhr
Статус: ACTIVE
Алгоритм: AES_256

2. Создание бакета и загрузка статического сайта
Был создан бакет static-site-1778333279 и настроен как статический веб-сайт(09.png,091.png, ).

# Создание бакета
s3cmd mb s3://static-site-1778333279 (03.png, 08.png)

# Включение хостинга
s3cmd ws-create s3://static-site-1778333279 --ws-index=index.html --ws-error=error.html

# Загрузка файлов сайта
s3cmd put index.html error.html s3://static-site-1778333279/
3. Настройка HTTPS
Сайт доступен по защищенному протоколу с использованием стандартного сертификата Yandex Cloud для доменов *.website.yandexcloud.net.

Адрес сайта: https://static-site-1778333279.website.yandexcloud.net

4. Настройка шифрования бакета
Для бакета было включено шифрование по умолчанию с использованием созданного KMS-ключа.

# Включение шифрования 
yc storage bucket update --name static-site-1778333279 --encryption key-id=abj0db3sjm76h6icdbhr
Проверка шифрования
Старые файлы: SSE: none
Новые файлы: SSE: aws:kms

Команда для проверки:
s3cmd info s3://static-site-1778333279/новый_файл.txt | grep SSE

5. Проверка доступности и безопасности
HTTPS доступ: curl -I https://static-site-1778333279.website.yandexcloud.net → HTTP/2 200 (05.png, 092.png)

Замочек в браузере: При открытии сайта в браузере в адресной строке отображается значок замка (SSL/TLS сертификат).(07.png, 093.png)

Итог
Статический сайт успешно развернут и доступен по HTTPS. Содержимое бакета шифруется с помощью KMS-ключа с алгоритмом AES-256, что обеспечивает защиту данных на стороне облачного хранилища.(093.png)
